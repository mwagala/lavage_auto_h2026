Demarrage Docker recommande
===========================

Depuis la racine du projet:

docker compose up --build

Cette commande demarre:
- PostgreSQL 16
- Redis 7
- bootstrap schema + donnees demo si la base est vide
- Flask
- Celery worker
- Celery Beat

Application:
- http://127.0.0.1:5000

Commandes utiles:

docker compose ps
docker compose logs -f web
docker compose exec postgres psql -U postgres -d lavage_auto
docker compose exec redis redis-cli ping
docker compose down
docker compose down -v

Variables utiles:

docker compose -p lavage-auto-dev up --build
APP_PORT=5050 docker compose up --build
BOOTSTRAP_AUTO_SEED=false docker compose up --build
BOOTSTRAP_RESET_DEMO_DATA=true docker compose up --build bootstrap

PowerShell:

$env:APP_PORT="5050"; docker compose up --build
$env:BOOTSTRAP_AUTO_SEED="false"; docker compose up --build

Alternative persistante:

Copier .env.example vers .env puis adapter APP_PORT ou les variables BOOTSTRAP_*.

Notes:
- Compose ne fixe ni container_name ni nom d'image applicative.
- Utiliser -p permet d'isoler plusieurs installations ou clones sur la meme machine.
- Seul le port applicatif est expose sur l'hote par defaut.
- PostgreSQL et Redis restent dans le reseau Docker pour eviter les conflits locaux.
- docker compose down conserve les volumes PostgreSQL et Redis.
- docker compose down -v supprime les volumes et repart de zero.
- Le dossier docs est volontairement exclu de l'image Docker.
